# 混合搜索模式使用指南

## 📋 方案概述

**核心思路**：脚本定义搜索需求 + 对话触发搜索执行

- **脚本职责**：定义搜索范围、生成任务清单、处理结果
- **AI 职责**：使用 WebSearch 工具执行搜索、返回结构化结果
- **优势**：绕过反爬机制、利用 AI 搜索能力、保持脚本灵活性

---

## 🚀 工作流程

```
步骤 1: 运行任务生成脚本
  ↓
步骤 2: 在对话框中触发 AI 搜索
  ↓
步骤 3: AI 填充搜索结果文件
  ↓
步骤 4: 运行结果处理脚本
  ↓
步骤 5: 生成最终早报 JSON
```

---

## 📝 详细操作步骤

### 步骤 1：生成搜索任务清单

```bash
python generate_search_tasks.py
```

**输出**：
- `search_tasks.json` - 包含 45 个搜索任务（9 个事业部 × 5 个维度）
- 时间窗口：T-2 到 T（例如：2026-04-08 至 2026-04-10）

**任务结构示例**：
```json
{
  "division_id": "czly",
  "division_name": "常州锂源",
  "task_type": "政策",
  "query": "常州锂源 磷酸铁锂 正极材料 政策 标准 补贴 监管",
  "priority": "high",
  "search_id": "czly_policy"
}
```

---

### 步骤 2：在对话框中触发搜索

**在对话框中输入**：
```
请执行搜索任务
```

**AI 将执行**：
1. 读取 `search_tasks.json` 文件
2. 使用 WebSearch 工具搜索 45 个关键词
3. 过滤旧闻（仅保留 T-2 到 T 的内容）
4. 填充 `search_results.json` 文件

**搜索结果格式**：
```json
{
  "常州锂源 磷酸铁锂 正极材料 政策 标准 补贴 监管": [
    {
      "title": "新闻标题",
      "content": "新闻摘要",
      "source": "来源媒体",
      "url": "原文链接",
      "date": "2026-04-09",
      "impact": "对事业部影响分析"
    }
  ]
}
```

---

### 步骤 3：处理搜索结果

```bash
python process_search_results.py
```

**处理逻辑**：
1. 加载 `search_tasks.json` 和 `search_results.json`
2. 按事业部分组（9 个事业部）
3. 按维度归类（政策/竞品/客户/前沿/市场）
4. 生成头条、主线判断、风险提示、小结
5. 输出最终早报 JSON

**输出文件**：
- `reports/2026-04-10_ai_search.json`

## ⚙️ 配置文件说明

### search_tasks.json

**字段说明**：
- `generated_at`: 任务生成时间
- `total_tasks`: 任务总数（45）
- `time_window`: 搜索时间窗口
- `divisions_count`: 事业部数量（9）
- `tasks`: 任务列表

**每个任务包含**：
- `division_id`: 事业部简称
- `division_name`: 事业部全称
- `task_type`: 维度类型（政策/竞品/客户/前沿/市场）
- `query`: 搜索关键词
- `priority`: 优先级（high/medium）
- `time_window`: 时间窗口
- `search_id`: 搜索任务 ID

## 💡 常见问题

### Q1: 为什么不用传统爬虫？
**A**: 百度、搜狗等主流搜索引擎有严格的反爬机制，传统爬虫容易被封禁 IP，且需要处理验证码。混合模式利用 AI 的 WebSearch 工具，天然绕过反爬。

### Q2: 搜索结果的时效性如何保证？
**A**: 脚本生成时会指定时间窗口（T-2 到 T），AI 搜索时会严格过滤，只保留时间窗口内的内容。

### Q3: 如何验证搜索结果的质量？
**A**: 每条结果都包含 `source`（来源媒体）、`url`（原文链接）、`date`（发布日期），可追溯验证。

### Q4: 搜索任务可以自定义吗？
**A**: 可以！修改 `generate_search_tasks.py` 中的 `DIVISIONS` 配置即可自定义事业部、关键词、维度等。

### Q5: 如果 AI 搜索失败怎么办？
**A**: 检查 `search_results.json` 是否为空。如果为空，可重新运行步骤 2，或在对话框中手动指定搜索关键词。

## 🔧 高级配置

### 自定义时间窗口

修改 `generate_search_tasks.py` 中的 `generate_search_tasks()` 函数：

```python
# 默认：T-2 到 T
date_range = f"{(today - timedelta(days=2)).strftime('%Y-%m-%d')} 至 {today.strftime('%Y-%m-%d')}"

# 自定义：T-5 到 T（更宽时间窗口）
date_range = f"{(today - timedelta(days=5)).strftime('%Y-%m-%d')} 至 {today.strftime('%Y-%m-%d')}"
```

### 自定义搜索维度

添加新的维度类型：

```python
division_tasks = [
    {
        "task_type": "政策",  # 现有维度
        "query": "...",
    },
    {
        "task_type": "供应链",  # 新增维度
        "query": "...",
    }
]
```

### 自定义优先级规则

```python
# 根据关键词重要性动态设置优先级
if "价格" in query or "订单" in query:
    priority = "high"
else:
    priority = "medium"
```
