### 2.2 JSON数据格式规范

**⚠️ 关键：格式错误会导致展示异常（空白、404等）**

#### 标准结构
```json
{
  "date": "2026-05-22",
  "window_start": "2026-05-19",
  "window_end": "2026-05-22",
  "departments": {
    "czly": {
      "name": "常州锂源事业部",
      "sections": [
        {
          "dim": "topnews",           // ⚠️ 必须是英文！
          "title": "今日关注",
          "items": [
            {
              "title": "标题",
              "content": "内容",
              "priority": "P0",
              "source": "来源",
              "date": "2026-05-22"
            }
          ]
        },
        {
          "dim": "market",           // ⚠️ 英文：market, policy, enterprise, tech, project, tips
          "title": "市场动态",
          "items": [/*...*/]
        }
      ]
    }
  }
}
```

#### ⚠️ 格式检查清单
1. **`dim` 值必须是英文**（不是中文！）
   - ✅ 正确：`"dim": "market"`
   - ❌ 错误：`"dim": "市场行情"`

2. **`sections` 必须是数组**（不是对象！）
   - ✅ 正确：`"sections": [{...}, {...}]`
   - ❌ 错误：`"sections": {"market": {...}, "policy": {...}}`

3. **`items` 必须是数组**（每个section下）
   - ✅ 正确：`"items": [{"title": "...", "content": "..."}]`
   - ❌ 错误：`"items": "字符串"`

4. **`content` 字段**：
   - 对于 `market` 维度的价格类条目，`content` 格式应为：`"价格：xx，涨跌：xx。来源：xx"`
   - 前端会自动解析为表格

---

### 2.3 前端展示规范

#### index_v3.html（首页）
- **嵌入模式**：`window.__EMBEDDED__` 必须定义在 `<head>` 内（第14行附近），在 `initDynamicData()` 执行之前
- **版本号**：每次更新必须修改 `<meta name="html-version">` 内容，强制浏览器刷新缓存
- **market维度**：必须显示为表格（品种 | 价格 | 涨跌 | 来源），不是折叠卡片

#### dept-archive.html（历史数据页）
- **版本号**：每次更新必须修改 `<meta name="html-version">` 内容
- **market维度**：必须显示为表格

## 三、更新早报的标准流程

### 场景1：更新今天的早报（日常操作）

#### 步骤1：生成/获取原始早报数据
```bash
# 运行转换脚本，生成 reports/YYYY-MM-DD.json
python convert_9bu_to_system.py
```

#### 步骤2：验证JSON数据格式
```python
# 检查脚本（必须运行！）
import json
with open('reports/2026-05-22.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 检查1：事业部ID是否正确
expected_ids = ['czly', 'lpsd', 'sdmd', 'felt', 'dhx', 'lhy', 'kls', 'sjld', 'bych']
actual_ids = list(data['departments'].keys())
print('事业部ID:', actual_ids)
assert all(id in expected_ids for id in actual_ids), "ID错误！"

# 检查2：dim值是否是英文
for dept_id, dept in data['departments'].items():
    for sec in dept.get('sections', []):
        assert sec.get('dim') in ['topnews', 'market', 'policy', 'enterprise', 'tech', 'project', 'tips'], \
            f"{dept_id} 的 dim 值错误: {sec.get('dim')}"

# 检查3：sections是否是数组
for dept_id, dept in data['departments'].items():
    assert isinstance(dept.get('sections'), list), f"{dept_id} 的 sections 不是数组！"
```

#### 步骤3：更新 index_v3.html 嵌入数据
1. 打开 `index_v3.html`
2. 找到第14行附近的 `window.__EMBEDDED__ = {...}`
3. 将 `report` 字段替换为新的JSON数据（注意转义引号）
4. 更新 `today` 字段为今天日期

#### 步骤4：更新版本号（强制缓存刷新）
- `index_v3.html`：修改第9行 `<meta name="html-version" content="YYYYMMDD_HHMMSS">`
- `dept-archive.html`：修改第6行 `<meta name="html-version" content="YYYYMMDDx">`

#### 步骤5：测试验证
1. **硬刷新**（`Ctrl+Shift+R`）首页
2. 点击事业部卡片 → "早报详情"，检查：
   - 今日关注：显示标题列表
   - 市场/价格：显示为表格
   - 专属提示：显示机会/风险/行动建议/重点关注
3. 打开 `dept-archive.html`，选择事业部+日期，检查：
   - 市场/价格：显示为表格

### 场景2：修复历史数据展示问题

#### 常见问题排查

| 问题 | 可能原因 | 修复方法 |
|------|---------|---------|
| 早报详情显示旧数据（如4月15日） | `window.__EMBEDDED__` 位置错误 | 确保定义在 `<head>` 内，在 `initDynamicData()` 之前 |
| 点击卡片后404加载 `reports/YYYY-MM-DD.json` | `getLatestReportForBU()` 未优先使用嵌入数据 | 检查函数开头是否有 `if (window.__EMBEDDED__)` 判断 |
| 早报详情内容空白 | `dim` 值是中文，导致 `sectionsToHTML()` 匹配失败 | 确保 `dim` 是英文（`market` 不是 `市场行情`） |
| 今日关注/专属提示显示"暂无内容" | `buildBriefSection()`/`buildTipsSection()` 读取 `section.content`（空），而不是 `section.items` | 修改函数，从 `items` 数组读取 |
| 市场/价格显示为折叠卡片，不是表格 | `renderItem()` 未识别价格类条目 | 在 `sectionsToHTML()` 中添加 `sec.dim === 'market'` 特殊处理 |
