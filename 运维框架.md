### 2.2 JSON数据格式规范

**⚠️ 关键：格式错误会导致展示异常（空白、404等）**

#### 标准结构
```json
{
  "date": "2026-05-22",
  "window_start": "2026-05-19",
  "window_end": "2026-05-22",
  "departments": {
    "czly": {
      "name": "常州锂源事业部",
      "sections": [
        {
          "dim": "topnews",           // ⚠️ 必须是英文！
          "title": "今日关注",
          "items": [
            {
              "title": "标题",
              "content": "内容",
              "priority": "P0",
              "source": "来源",
              "date": "2026-05-22"
            }
          ]
        },
        {
          "dim": "market",           // ⚠️ 英文：market, policy, enterprise, tech, project, tips
          "title": "市场动态",
          "items": [/*...*/]
        }
      ]
    }
  }
}
```

#### ⚠️ 格式检查清单
1. **`dim` 值必须是英文**（不是中文！）
   - ✅ 正确：`"dim": "market"`
   - ❌ 错误：`"dim": "市场行情"`

2. **`sections` 必须是数组**（不是对象！）
   - ✅ 正确：`"sections": [{...}, {...}]`
   - ❌ 错误：`"sections": {"market": {...}, "policy": {...}}`

3. **`items` 必须是数组**（每个section下）
   - ✅ 正确：`"items": [{"title": "...", "content": "..."}]`
   - ❌ 错误：`"items": "字符串"`

4. **`content` 字段**：
   - 对于 `market` 维度的价格类条目，`content` 格式应为：`"价格：xx，涨跌：xx。来源：xx"`
   - 前端会自动解析为表格

### 2.3 前端展示规范

#### index_v3.html（首页）
- **嵌入模式**：`window.__EMBEDDED__` 必须定义在 `<head>` 内（第14行附近），在 `initDynamicData()` 执行之前
- **版本号**：每次更新必须修改 `<meta name="html-version">` 内容，强制浏览器刷新缓存
- **market维度**：必须显示为表格（品种 | 价格 | 涨跌 | 来源），不是折叠卡片

#### dept-archive.html（历史数据页）
- **版本号**：每次更新必须修改 `<meta name="html-version">` 内容
- **market维度**：必须显示为表格

### 8.2 下拉菜单交互逻辑（JS 延迟定时器）

**问题**：纯 CSS `:hover` 无法精确控制显示时机，导致鼠标移入时菜单已消失，难以选中。

```javascript
// 位于 renderBuTabs() 内部
let buHideTimer = null;
const BU_HIDE_DELAY = 200;

bar.querySelectorAll('.bu-tab-wrap').forEach(wrap => {
  const menu = wrap.querySelector('.bu-dropdown');

  // ① 进入 Tab 区 → 立即显示，关闭其他菜单
  wrap.addEventListener('mouseenter', function() {
    clearTimeout(buHideTimer);
    bar.querySelectorAll('.bu-dropdown').forEach(m => m.style.display = 'none');
    if (menu) menu.style.display = 'block';
  });

  // ② 离开 Tab 区 → 启动延迟隐藏
  wrap.addEventListener('mouseleave', function() {
    buHideTimer = setTimeout(() => {
      if (menu) menu.style.display = 'none';
    }, BU_HIDE_DELAY);
  });

  // ③ 进入下拉菜单 → 取消隐藏定时器
  if (menu) {
    menu.addEventListener('mouseenter', function() {
      clearTimeout(buHideTimer);
      buHideTimer = null;
    });

    // ④ 离开下拉菜单 → 重新启动延迟隐藏
    menu.addEventListener('mouseleave', function() {
      buHideTimer = setTimeout(() => {
        if (menu) menu.style.display = 'none';
      }, BU_HIDE_DELAY);
    });
  }
});
```

### 8.3 时间倒序展示逻辑（核心规则）

**所有新闻条目必须按时间倒序（最新优先）展示**，在 `renderDimContent()` 中实现：

```javascript
const order = { P0:0, P1:1, P2:2 };
const sorted = [...items].sort((a, b) => {
  // ① 日期倒序优先
  const dateA = (a._date || a.date || '').replace(/\//g, '-');
  const dateB = (b._date || b.date || '').replace(/\//g, '-');
  if (dateA !== dateB) return dateB.localeCompare(dateA);
  // ② 同日按优先级正序（P0 > P1 > P2）
  return (order[a.priority] ?? 2) - (order[b.priority] ?? 2);
});
```

**排序优先级**：日期倒序 → 优先级正序（P0 最前，P2 最后）  
**字段兼容**：`a._date`（内嵌数据来源）优先，`a.date` 为兜底

### 8.4 维度分类规范（新闻归类逻辑）

分类在两个层级生效：① 早报生成时 `dim` 字段写入 JSON ② 前端 `normalizeDim()` 解析路由。两级须保持一致。

#### 四个维度定义

| 维度 | 标准 key | 含义 | 示例 |
|------|---------|------|------|
| 宏观新闻 | `policy` | 政策、法规、补贴、监管 | 工信部发文、碳中和政策、补贴通知 |
| 行业新闻 | `market` | 供需、库存、行情、市场分析 | 磷酸铁锂涨价分析、锂矿现货下跌 |
| 企业动态 | `competitor` | 竞品/竞争对手、企业动态、项目招标 | 宁德时代扩产、某企业获得订单 |
| 技术产品 | `frontier` | 技术进展、研发、产品创新 | 新款电芯发布、新工艺突破 |

#### 分类匹配顺序（`normalizeDim()` 执行逻辑）

**第一步：精确匹配 `DIM_MAP`**（直接命中，返回标准 key）

```javascript
const DIM_MAP = {
  // 英文精确 key（早报 JSON 格式）
  'market':     'market',
  'policy':     'policy',
  'competitor': 'competitor',
  'frontier':   'frontier',
  'enterprise': 'competitor',   // 早报格式 → 企业动态
  'tech':       'frontier',     // 早报格式 → 技术产品
  'customer':   'competitor',
  'project':    'competitor',
  // 中文精确 key
  '政策':       'policy',
  '竞品':       'competitor',
  '市场':       'market',
  '前沿':       'frontier',
};
```

**第二步：模糊匹配**（精确未命中时，按标题关键词兜底）

```javascript
if (key.includes('政策') || key.includes('条例') || key.includes('补贴')
  || key.includes('通知') || key.includes('发文') || key.includes('进出口')
  || key.includes('监管') || key.includes('法规') || key.includes('双碳')
  || key.includes('碳中和') || key.includes('新能源战略'))
  return 'policy';

if (key.includes('市场') || key.includes('价格') || key.includes('行情')
  || key.includes('供需') || key.includes('库存'))
  return 'market';

if (key.includes('企业') || key.includes('竞品') || key.includes('竞争')
  || key.includes('动态') || key.includes('项目') || key.includes('招标')
  || key.includes('客户'))
  return 'competitor';

if (key.includes('技术') || key.includes('产品') || key.includes('前沿')
  || key.includes('研发'))
  return 'frontier';
```

**第三步：未匹配 → 丢弃 + 控制台警告**（返回 `null`，该条目不会展示，同时 F12 控制台打印 `[行业新闻] 未知维度(×格式)，已丢弃: xxx`）

> ⚠️ **已知风险**：若早报生成侧使用了未在关键词列表中的 dim 值（如"氢能动态""储能招标"），该 section 下所有条目会被静默丢弃。控制台警告可以帮助发现这类情况，后续在 `DIM_MAP` 或模糊关键词中添加对应分类即可修复。

#### 早报生成时的 dim 写入规范

在生成 `reports/YYYY-MM-DD.json` 时，**每个 section 必须指定 `dim` 字段**（英文标准 key），参考上表分类。不得使用中文 dim 值（如"政策风向"），中文仅作 label 展示用。

#### 常见错误

| 错误写法 | 正确写法 | 原因 |
|---------|---------|------|
| `{"dim": "政策风向"}` | `{"dim": "policy"}` | 前端模糊匹配可能失效 |
| `{"dim": "竞品动态"}` | `{"dim": "competitor"}` | 未在 DIM_MAP 中，会被丢弃 |
| `{"dim": "前沿动态"}` | `{"dim": "frontier"}` | 前端只认 `frontier` |
| `{"dim": "topnews"}` | `{"dim": "policy"}`（或其他合适维度） | topnews 本身非展示维度 |

兼容中英文 dim 值（旧数据/新数据均可正确路由）。

### 8.6 价格表过滤函数

```javascript
// 判断一条早报条目是否为价格表格形式（由其他模块展示，此处过滤）
function isPriceTableEntry(item) {
  const content = item.content || item.title || '';
  if (content.includes('价格：') && content.includes('涨跌：')) {
    return true;  // 过滤，不纳入行业新闻
  }
  return false;   // 保留
}
```

**判断逻辑**：同时含"价格："和"涨跌："的条目，判定为价格表格，不纳入行业新闻。

**效果对比**：
- 过滤掉：`"价格：183,250元/吨，涨跌：+5,250元/吨。来源：光大期货"`
- 保留：`"铱价年初至今疯涨近六成，铱现货价从2025年12月约1168元/克升至2026年4月约1852元/克……"`（含价格数字但非表格格式）

## 九、今日热点资讯更新规范

### 9.1 功能概述

**目的**：在首页（index_v3.html）展示5条热点新闻轮播，数据来自9个事业部当天早报的"今日关注"部分。

**核心特点**：
- 数据文件：`hot_news_data.json`（由自动化任务生成）
- 图片来源：Unsplash API（浏览器端调用，为每条新闻配图）
- 更新时间：每个工作日早上9:30（自动化任务触发）
- 容错机制：当天早报未更新时，沿用前一天数据

**数据流向**：
```
9个事业部早报JSON (reports/*.json)
    ↓
generate_hot_news.py (均匀抽取5条)
    ↓
hot_news_data.json (新闻列表，无图片URL)
    ↓
index_v3.html ← loadHotNews() 加载JSON
    ↓
浏览器调用 Unsplash API ← 为每条新闻配图
    ↓
展示：新闻轮播 (5s自动切换)
```

### 9.2 更新逻辑

#### 数据抽取规则
1. **来源**：9个事业部当天早报的 `sections["今日关注"]` 数组
2. **数量**：5条新闻
3. **均匀抽取算法**：
   - 第一轮：每个事业部最多选1条（随机选择事业部顺序）
   - 第二轮：如果不够5条，允许某些事业部选2条（避免单事业部占多数）
4. **容错**：检查事业部JSON的 `header["报告时间"]` 字段
   - 如果是今天 → 生成新数据
   - 如果不是今天 → 沿用昨天的 `hot_news_data.json`（不覆盖）

#### 图片配图逻辑
- **位置**：浏览器端（index_v3.html 的 `loadHotNews()` 函数）
- **API**：Unsplash API（`https://api.unsplash.com/search/photos`）
- **流程**：
  1. 从 `hot_news_data.json` 加载新闻列表
  2. 对每条新闻，调用 `extractKeywords(title)` 提取英文搜索关键词
  3. 调用 `fetchUnsplashImage(query, excludeUrls)` 获取图片URL
  4. 设置轮播幻灯片的背景图（`slide.querySelector('.news-slide-bg').style.backgroundImage`）
- **防重复机制**：`excludeUrls` 参数记录已使用的图片URL，避免5张图片有重复

#### 更新时间
- **自动化任务**：每个工作日早上9:30（Monday-Friday, 09:30）
- **任务名称**：`hot-news-generator`（WorkBuddy自动化）
- **执行内容**：
  1. 运行 `generate_hot_news.py` 脚本
  2. 如果成功，提交 `hot_news_data.json` 到Git并推送（触发GitHub Pages部署）
  3. 如果失败，不覆盖现有数据

### 9.4 容错机制

#### 容错1：事业部早报未更新

**检测逻辑**：
- 脚本检查9个事业部JSON的 `header["报告时间"]` 字段
- 如果有任何一个不是今天 → 判定为"未更新"

- 控制台输出：`[信息] 部分事业部未更新（X/9），沿用昨天的热点新闻数据`
- 脚本退出码：0（成功，但未生成新数据）

**影响**：
- 首页展示的是昨天的热点新闻（不会空白）
- 用户不会感知到数据中断

#### 容错2：所有事业部都没有"今日关注"新闻

**检测逻辑**：
- 脚本提取所有事业部的 `sections["今日关注"]`，如果数组为空

**处理方式**：
- 生成默认数据：`[{"bu": "系统", "title": "暂无今日关注数据，请先生成早报"}]`
- 控制台输出：`[错误] 没有找到任何今日关注新闻，使用默认数据`

**影响**：
- 首页展示默认提示（不会崩溃）
- 用户看到"暂无今日关注数据，请先生成早报"

#### 容错3：Unsplash API调用失败

**检测逻辑**：
- 浏览器端调用 `fetchUnsplashImage()` 失败时（网络错误、API限制等）

**处理方式**：
- 捕获异常，控制台输出：`[热点资讯] Unsplash图片加载失败: [错误详情]`
- 该幻灯片不设置背景图（显示默认渐变背景）

**影响**：
- 新闻轮播正常显示（只是没有配图）
- 用户看到文字新闻+默认背景
