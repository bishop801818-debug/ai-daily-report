- ✅ JSON 结构化数据（`常州锂源早报_20260408.json`）

## 🚀 生成流程

### 第一步：运行搜索脚本

我已经创建了简化版搜索脚本 `run_search.py`，执行以下命令：

```bash
python run_search.py
```

**脚本功能**：
- 自动读取 5 个模块的搜索关键词（政策/竞品/客户/前沿/市场）
- 自动添加日期提示词（4 月 6 日 OR 4 月 7 日 OR 4 月 8 日）
- 并行调用 Tavily API 搜索最新信息
- 输出 `briefing_search_results.json` 文件

**搜索结果示例**：

### 第二步：读取 JSON 数据

搜索完成后，会生成 `briefing_search_results.json` 文件，包含：
- 5 个模块的搜索结果
- 每条结果的标题、内容、URL、来源域名、相关度评分

### 第三步：按 Skill 规范生成早报

根据 `lithium-daily-briefing.md` 中的规则，进行以下处理：

#### 3.1 日期过滤（T 到 T-2）
- 今日：2026-04-08
- 数据窗口：2026-04-06 ~ 2026-04-08
- 排除旧闻（标题含"回顾/盘点/总结"，正文含"去年/早在"等）

#### 3.2 重要性判断（A/B/C/D 四级）

**A 级（头条）**：
- ✅ 宁德时代 M3P 电池"上车"首发（LMFP 商业化加速）

**B 级（板块头条）**：
- ✅ 2025 年储能电池出货量 652GWh（+76%）
- ✅ 碳酸锂价格持续反弹（15.85 万元/吨）
- ✅ 德方纳米第四代高压密 LFP 批量出货

**C 级（板块正文）**：
- ✅ 吉林省零碳园区政策
- ✅ 3 月新能源乘用车批发量 112 万辆

#### 3.3 三段式摘要写作

每条信息按以下格式输出：
```
【事件名称】
[发生了什么，含数据 + 时间]
[为什么重要]
[对事业部意味着什么]
来源：[domain]
```

#### 3.4 经营判断生成

**示例**：
> ❌ 资讯转述："某公司发布新品，行业发展向好。"
> 
> ✅ 经营判断："宁德时代 M3P 电池 Q3 首发，意味着 LMFP 技术从实验室走向市场，对传统 LFP 形成技术升级压力。常州锂源高压实 LFP 的溢价窗口进一步收窄，需加速 LMFP 中试线布局。"

## 📊 生成的早报结构

### Markdown 早报（9 大板块）

1. **🔥 头条聚焦** - 当天对事业部影响最大的动态
2. **📋 今日主线判断** - 2-4 句话总结核心变化
3. **📜 政策风向** - 政策动态及影响
4. **📌 客户动态** - 核心客户采购/项目进展
5. **⚔️ 竞品动态** - 竞品产能/技术/客户变化
6. **📊 市场行情** - 价格 + 供需动态
7. **💡 前沿技术** - 技术升级/替代风险
8. **🎯 行动建议** - 机会 + 风险 + 具体行动
9. **📝 数据质量说明** - 搜索情况 + 信息量评估

### JSON 结构化数据

用于页面渲染，包含：
- `division_id`: 事业部代码
- `report_date`: 报告日期
- `lead_judgment`: 主线判断
- `headline`: 头条聚焦
- `sections`: 各板块内容（policy/customer/competitor/market/frontier）
- `action`: 行动建议（opportunity/risk）
- `data_quality`: 数据质量说明

## 🎯 关键成功要素

### 1. 搜索关键词质量
- 每个模块 3-5 个关键词
- 必须附加日期提示词
- 示例：`磷酸铁锂 政策 动力电池 政策 4 月 6 日 OR 4 月 7 日 OR 4 月 8 日`

### 2. 日期过滤规则
- 页面 HTML 提取日期在 T 到 T-2 范围内 → 通过
- 正文含"4 月 6 日/7 日/8 日/今日/昨天" → 新鲜信号
- 正文含"去年/早在/月份以来" → 旧闻信号，排除

### 3. 重要性判断框架

**三问法**：
1. **谁？** - 客户/竞品/行业整体
2. **有多大？** - 产能多少万吨？价格涨跌多少？
3. **对我们意味着什么？** - 直接利好/利空/中性

### 4. 内容写作规则
- ✅ 写经营含义，不写新闻复述
- ✅ 含数据（价格/产量/份额/时间）
- ✅ 短句快扫（每段不超过 3 句话）
- ✅ 注明来源（域名）
- ❌ 禁止空话（"行业态势良好"）

## 🔄 日常使用流程

**每天早晨执行**：

1. **运行搜索**（5 分钟）
   ```bash
   python run_search.py
   ```

2. **读取 JSON**（1 分钟）
   - 打开 `briefing_search_results.json`
   - 快速浏览各模块候选结果数量

3. **生成早报**（10-15 分钟）
   - 按重要性分级（A/B/C/D）
   - 写三段式摘要
   - 生成经营判断
   - 推导行动建议

4. **输出归档**（2 分钟）
   - 保存 Markdown 早报
   - 保存 JSON 数据
   - 更新索引文件

## 💡 进阶技巧

### 场景 A：价格反弹
**关键词**：涨价、反弹、上行、持续上涨
**分析框架**：
- 反弹幅度多少？（查具体数字）
- 反弹持续多久？（是否超 2 周）
- 是否传导到 LFP 材料环节？
- **结论**：量价齐升 vs 仅成本推动利润承压

### 场景 B：大客户份额攻防
**关键词**：宁德时代、采购、份额、供应
**分析框架**：
- 总量变化多少？（增量 vs 存量再分配）
- 谁在抢份额？（竞品是否在列）
- 我方是否在增份额名单内？
- **结论**：我方受益 vs 存在替代风险

### 场景 C：竞品产能变化
**关键词**：投产、扩产、检修、停产
**分析框架**：
- 规模多大？（万吨/月）
- 持续多久？（检修时长）
- 影响哪些客户？
- **结论**：填补机会窗口 vs 供给侧压力加剧

### 场景 D：LMFP 进展
**关键词**：LMFP、磷酸锰铁锂、新品、量产
**分析框架**：
- 是否有车企定点？（有定点 = 进展加速）
- 规模多大？（亿投资）
- 是否有工艺壁垒？（专利/独家工艺）
- **结论**：行业升级加速，我方跟进节奏是否匹配

## ⚠️ 常见问题

### Q1: 搜索结果太少怎么办？
**A**: 
- 检查关键词是否太窄，减少到 3 个核心词
- 扩大日期范围到 T-3 或 T-4
- 增加同义词（如"LFP"和"磷酸铁锂"）

### Q2: 如何判断信息真伪？
**A**: 
- 优先使用 1 级来源（公司公告/互动易/官方公众号）
- 交叉验证（同一事件多个来源报道）
- 标注"待核实"（对单一来源的重要信息）

### Q3: 行动建议太泛化怎么办？
**A**: 
- 必须有具体对象（找谁、做什么）
- 有时间边界（本周/本月/下次会议前）
- 有量化参考（多少量/多少钱）
- ❌ 禁止："建议关注""建议持续跟踪"
