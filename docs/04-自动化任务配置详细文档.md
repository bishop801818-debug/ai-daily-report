## 📋 概述

本平台的自动化任务分为两层：

| 层级 | 调度方式 | 任务数量 | 用途 |
|------|---------|---------|------|
| **WorkBuddy 自动化** | WorkBuddy client 定时触发 | 6 个 | 早报下载/数据抓取/热点新闻生成 |
| **Windows 任务计划程序** | 系统级定时任务 | 1 个 | Python 脚本定时执行 |

> WorkBuddy 自动化的详细故障案例见 [05-常见错误与排障详细文档](05-常见错误与排障详细文档.md) 第三节。

## 一、WorkBuddy 自动化任务概览

WorkBuddy 是主要的自动化调度平台，位于 `D:\buddy\` 目录下。任务配置在 `.workbuddy/automations/` 中。

| 任务名称 | 频率 | 用途 |
|---------|------|------|
| 早报 ZIP 下载与更新 | 每日 | 从飞书下载 MD 文件 → 生成 `reports/YYYY-MM-DD.json` |
| 碳酸锂期货 K 线抓取 | 每日 | `akshare` 抓取 LC0 主连数据 → `market_lc.json` |
| 锂辉石+锂云母价格抓取 | 每日 17:00 | 价格抓取 → `data/` 目录 |
| 锂矿价格历史数据更新 | 每日 | 更新 `data/lithium_ore_price_history.json` 等 |
| 尿素期货数据抓取 | 每日 | WTI 原油 + 尿素期货 → `data/` |
| 热点新闻生成 | 工作日 9:30 | 生成 `hot_news_data.json` |

## 二、Windows 任务计划程序设置

### 📋 目标
通过 Windows 任务计划程序定时运行 Python 数据更新脚本。

---

## 🚀 设置步骤

### 步骤1：打开任务计划程序
1. 按 `Win + R` 打开运行对话框
2. 输入 `taskschd.msc` 并按回车
3. 任务计划程序窗口将打开

### 步骤2：创建基本任务
1. 在右侧操作面板中，点击 **"创建基本任务"**
2. 任务名称：`每日市场数据更新`
3. 描述：`每天自动更新 market_lc.json 和 market_lfp.json`
4. 点击 **"下一步"**

### 步骤3：设置触发器
1. **希望该任务何时开始？** 选择 **"每天"**
2. 点击 **"下一步"**
3. **开始时间**：设置为 **每天上午 9:00**（或您希望的时间）
4. **开始日期**：保持默认（今天）
5. 点击 **"下一步"**

### 步骤4：设置操作
1. **希望该任务执行什么操作？** 选择 **"启动程序"**
2. 点击 **"下一步"**
3. **程序或脚本**：选择需要定时执行的 Python 脚本或批处理文件
4. **起始于（可选）**：`D:\trae\AI Daily report\`
5. 点击 **"下一步"**

### 步骤5：完成设置
1. 勾选 **"当单击完成时，打开此任务属性的对话框"**
2. 点击 **"完成"**

## ⚙️ 高级设置（重要）

在任务属性对话框中，进行以下设置：

### 1. 常规选项卡
- ✅ 勾选 **"不管用户是否登录都要运行"**
- ✅ 勾选 **"使用最高权限运行"**
- 配置为：**Windows 10** (或您的系统版本)

### 2. 触发器选项卡
- 双击触发器，设置：
  - ✅ **启用**
  - 重复间隔：**1天**
  - 持续时长：**无限期**

### 3. 操作选项卡
- 确认程序路径正确：`D:\trae\AI Daily report\update_data_daily.bat`

### 4. 条件选项卡
- ❌ **取消勾选** "只有在计算机使用交流电源时才启动此任务"（如果是笔记本）
- ✅ 勾选 "唤醒计算机运行此任务"

### 5. 设置选项卡
- ✅ 勾选 "如果任务失败，重新启动每隔：**10分钟**"
- ✅ 勾选 "如果请求后任务还在运行，强行将其停止：**1小时**"
- 如果任务已经运行，则应用规则：**"停止现有实例"**

### 6. 点击 **"确定"** 保存设置

## 🔧 高级设置详解（实战必读）

### 1. 工作目录设置（最容易出错！）

**问题**：任务计划程序运行时，**默认工作目录是 `C:\Windows\System32\`**，不是你的脚本目录！  
如果你的Python脚本用了相对路径（如 `open('data.json')`），会找不到文件。

**解决方案**：在 **"操作" 选项卡** 中：
1. 选中你的操作，点击 **"编辑"**
2. **"起始于(可选)"** 填写脚本所在目录，例如：
   ```
   D:\trae\AI Daily report\
   ```
3. **重要**：路径末尾必须有反斜杠 `\`，否则可能不生效

**验证方法**：在Python脚本开头加一行调试代码：
```python
import os
print("当前工作目录:", os.getcwd())
```

---

### 2. 运行账户选择策略

在 **"常规" 选项卡** 中，**"安全选项"** 区域：

| 选项 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **"只在用户登录时运行"** | 需要访问用户桌面、网络驱动器、用户环境变量 | 权限简单，调试方便 | 用户注销后任务停止 |
| **"不管用户是否登录都要运行"** | 服务器环境、需要7x24运行 | 稳定，不依赖用户会话 | 无法访问用户桌面和网络驱动器 |
| **"使用最高权限运行"** | 脚本需要管理员权限（如访问受限目录） | 权限充足 | 可能触发UAC提示（如果交互式运行） |

**推荐配置**：
- ✅ 勾选 **"不管用户是否登录都要运行"**
- ✅ 勾选 **"使用最高权限运行"**
- **注意**：如果脚本需要访问**网络共享盘**（如 `Z:\`），这个配置可能无法访问。解决方案：改用 **UNC路径**（如 `\\server\share\`）

---

### 3. 网络可用性条件

在 **"条件" 选项卡** 中：

**场景**：你的Python脚本需要访问网络（如下载文件、调用API）。

**设置**：
1. ✅ 勾选 **"只有在以下网络连接可用时才启动此任务"**
2. 选择 **"任何连接"** 或指定具体网络适配器
3. ✅ 勾选 **"唤醒计算机运行此任务"**（如果脚本需要在计算机休眠时运行）

**注意**：如果勾选了这个选项，但任务触发时网络还没准备好（如计算机刚唤醒），任务会**等待网络可用**后才运行，可能导致延迟。

---

### 4. 任务超时与重试机制

在 **"设置" 选项卡** 中：

| 设置项 | 推荐值 | 说明 |
|--------|---------|------|
| **"如果任务失败，重新启动每隔"** | 10分钟 | 任务失败后自动重试间隔 |
| **"如果请求后任务还在运行，强行将其停止"** | 1小时 | 防止任务卡死（如Python脚本陷入死循环） |
| **"如果任务已经运行，则应用规则"** | **"停止现有实例"** | 避免多个实例同时运行导致数据冲突 |
| **"如果任务无法运行，立即尝试重新运行"** | ✅ 勾选 | 任务触发失败后，立即重试一次 |

**高级：设置任务超时**
1. 在 **"设置" 选项卡** 中，点击 **"添加参数(可选)"**（某些Windows版本有此选项）
2. 或通过 **PowerShell** 设置：
   ```powershell
   $task = Get-ScheduledTask -TaskName "每日市场数据更新"
   $task.Settings.ExecutionTimeLimit = "PT1H"  # 超时1小时
   $task | Set-ScheduledTask
   ```

### 5. PowerShell 执行策略（如果任务运行PS脚本）

**问题**：Windows默认禁止运行PowerShell脚本（`.ps1`文件），会报错：  
`无法加载，因为在此系统上禁止运行脚本。`

**解决方案A**（推荐）：在 **"操作"** 中，不直接运行 `.ps1`，而是运行：
```
程序或脚本：powershell.exe
添加参数：-ExecutionPolicy Bypass -File "D:\trae\AI Daily report\script.ps1"
```

**解决方案B**：修改系统执行策略（需要管理员权限）：
```powershell
Set-ExecutionPolicy RemoteSigned -Scope LocalMachine
```

### 6. 查看详细日志（排障必备）

**任务计划程序自带日志**：
1. 在任务计划程序中，点击你的任务
2. 下方切换到 **"历史记录"** 选项卡
3. 可以看到每次运行的 **事件ID**：
   - **100** - 任务已启动
   - **101** - 任务已完成
   - **102** - 任务已失败
   - **103** - 任务已超时

**查看详细错误**：
- 点击某个事件，下方会显示 **"常规"** 和 **"详细信息"**
- **"详细信息"** 中的 **"事件数据"** 包含错误代码和描述

**启用任务计划程序日志**（如果"历史记录"为空）：
1. 打开 **"事件查看器"**（`eventvwr.msc`）
2. 导航到：**应用程序和服务日志 > Microsoft > Windows > TaskScheduler > Operational**
3. 右键点击 **"Operational"**，选择 **"启用日志"**

### 7. 环境变量与路径问题

**问题**：任务计划程序运行时，**环境变量可能与用户登录时不同**（如 `PATH`、`PYTHONPATH`）。

**排查方法**：在Python脚本中打印环境变量：
```python
import os
print("PATH:", os.environ.get('PATH'))
print("PYTHONPATH:", os.environ.get('PYTHONPATH'))
```

**解决方案**：
1. 在 **"操作"** 的 **"添加参数"** 中设置环境变量：
   ```
   程序或脚本：cmd.exe
   添加参数：/c "set PYTHONPATH=D:\mylibs && python script.py"
   ```
2. 或者在Python脚本开头手动添加路径：
   ```python
   import sys
   sys.path.append('D:\\mylibs')
   ```

---

### 8. 触发器高级设置（多个触发器、延迟启动）

**场景**：需要多个触发时间（如工作日9:00，周末10:00）。

**设置**：
1. 在 **"触发器" 选项卡** 中，点击 **"新建"**
2. 可以创建多个触发器，每个触发器独立设置：
   - 触发器1：每周，周一至周五，9:00
   - 触发器2：每周，周六、周日，10:00

**延迟启动**（避免任务同时运行）：
1. 编辑触发器，展开 **"高级设置"**
2. 勾选 **"延迟任务时间"**，设置为 **5分钟**
3. 这样任务触发后会等待5分钟再运行（适用于多个任务错峰运行）

## 🔧 故障排除

### 问题1：任务没有运行
- 检查任务是否 **"启用"**
- 检查触发器设置是否正确
- 查看 **"历史记录"** 中的错误信息

### 问题2：Python脚本执行失败
- 确认Python已添加到系统PATH
- 确认目标脚本能手动运行
- 检查依赖库是否已安装（如 `akshare`、`openpyxl` 等）

### 问题3：权限问题
- 确认任务设置为 **"使用最高权限运行"**
- 确认用户有访问 `D:\trae\AI Daily report\` 的权限
