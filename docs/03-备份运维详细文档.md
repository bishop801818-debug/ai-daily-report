## 📝 版本命名规则

### 备份文件名格式
```
文件名.后缀.golden_YYYY-MM-DD_HHMMSS_vNEW
```

### 版本号规则
- **HTML文件**：每个文件独立版本号
  - `index_v3.html`：v18, v19, v20...
  - `database_hub.html`：v3, v4, v5...
  - 其他HTML文件同理
  
- **数据库文件**：统一版本号（与主要HTML文件同步）
  - 通常跟随 `index_v3.html` 的版本号

### 版本号提升规则
- 每次完整备份，所有文件的版本号 +1
- 示例：当前 `index_v3.html` 是 v18，备份后变成 v19

## 🔄 回滚操作流程

### 回滚触发指令
用户发送以下任一指令：
- **"回滚到vXX"**
- **"恢复到vXX"**
- **"使用vXX版本"**

### 回滚执行步骤
1. 确认目标版本号
2. 查找对应的 `.golden_*` 备份文件
3. 复制备份文件到工作文件
4. 验证回滚结果
5. 回复用户：**"已回滚到vXX，请刷新浏览器确认"**

### 回滚脚本使用
```bash
# 双击运行回滚脚本
rollback_to_v17.bat
```
