## 三、数据源头：Excel → embedded JS

### 3.1 现状

`*_embedded_data.js` 文件是**所有数据库和看板的唯一权威数据源**。项目内**没有** Python 脚本能直接从 Excel 生成这些 JS 文件。它们来自外部渠道（飞书多维表格导出、Excel 手动转换等）。

### 3.2 embedded JS 文件结构

所有 JS 文件结构一致：

```javascript
const EMBEDDED_DATA = {   // 变量名因数据库而异
  "update_time": "2026-05-28 17:28:32",
  "source": "飞书多维表格-磷酸铁锂产业链数据库（Excel追加更新）",
  "tables": [
    {
      "table_name": "LFP-行业整体产量",
      "data": [
        { "月份": "2026-04", "月度产量": 46980, "产能利用率": "80%" },
        ...
      ]
    },
    ...
  ]
}
```

### 3.3 各数据库 JS 变量名对照

| 文件 | 变量名 |
|------|--------|
| `embedded_data.js` (LFP) | `EMBEDDED_DATA` |
| `carbonate_embedded_data.js` | `EMBEDDED_DATA` |
| `electrolyte_embedded_data.js` | `ELECTROLYTE_DATA` |
| `lib_battery_embedded_data.js` | `EMBEDDED_DATA` |
| `automotive_embedded_data.js` | `AUTOMOTIVE_DATA` |
| `recycling_embedded_data.js` | `EMBEDDED_DATA` |
| `ternary_embedded_data.js` | `TERNARY_DATA` |

### 3.4 特殊注意

- **LFP** 的文件名是 `embedded_data.js`（无 `lfp_` 前缀），容易混淆
- `embedded/embedded_data.js` 是旧版本（2026-05-18），根目录的是新版（2026-05-28），**更新时以根目录为准**
- `lfp_all_data.js` 是另一个 JS 数据文件，变量名也是 `EMBEDDED_DATA`，与 `embedded_data.js` 内容不同

## 四、数据库页面更新流程

### 4.1 前提

数据库页面（`*_data_v2.html`）是**动态渲染**的——它们加载 `*_embedded_data.js`，用 JavaScript 遍历 `tables` 数组生成 HTML 表格。因此：

> **只要更新了 `*_embedded_data.js`，数据库页面就自动展示最新数据，无需修改 HTML。**

### 4.2 更新步骤

```
1. 获取最新 Excel 数据源
2. 更新对应的 *_embedded_data.js 文件
3. 同步到 embedded/ 目录
4. 验证: http://localhost:8888/{prefix}_data_v2.html
```

### 4.3 各页面加载方式细节

| 数据库页面 | 数据加载方式 | 回退机制 |
|-----------|-------------|---------|
| `lfp_data_v2.html` | `<script src="embedded_data.js">` → `EMBEDDED_DATA.tables` | `fetch('lfp_all_data.json')` |
| `carbonate_data_v2.html` | `<script src="carbonate_embedded_data.js">` → `EMBEDDED_DATA.tables` | `fetch('carbonate_all_data.json')` |
| `electrolyte_data_v2.html` | `<script src="electrolyte_embedded_data.js">` → `ELECTROLYTE_DATA.tables` | 无回退 |
| `lib_battery_data_v2.html` | `<script src="lib_battery_embedded_data.js">` → `EMBEDDED_DATA.tables` | `fetch('lib_battery_all_data.json')` (但 JSON 不存在) |
| `automotive_data_v2.html` | `<script src="automotive_embedded_data.js">` → `AUTOMOTIVE_DATA` | 内联 HTML 备用行 |
| `recycling_data_v2.html` | `<script src="recycling_embedded_data.js">` → `EMBEDDED_DATA.tables` | `fetch('recycling_all_data.json')` (但 JSON 不存在) |
| `ternary_data_v2.html` | `<script src="ternary_embedded_data.js">` → `TERNARY_DATA.tables` | `fetch('ternary_all_data.json')` |

## 五、可视化看板更新流程

### 5.1 两种数据模式

#### 模式 A：动态加载（3 个看板）

**看板**: `electrolyte_charts.html`, `lib_battery_charts.html`, `ternary_charts.html`

这些看板直接加载 `*_embedded_data.js`，通过 `getTable()` 等辅助函数动态提取数据渲染图表。

**更新方式**：更新 embedded JS → 看板自动跟随，**无需修改 HTML**。

```javascript
// 示例 (electrolyte_charts.html)
var data = ELECTROLYTE_DATA;
function getTable(name) {
    return data.tables.find(t => t.table_name === name)?.data || [];
}
var industryData = getTable('电解液-行业整体产量');
// 动态提取月份、产量等字段 → 渲染 ECharts
```

#### 模式 B：硬编码数组（4 个看板）

**看板**: `lfp_charts.html`, `carbonate_charts.html`, `automotive_charts.html`, `recycling_charts.html`

这些看板在 HTML 中直接写了 `var LFP_MONTHS = [...]` 这样的 JS 数组，不读取 embedded JS。

**更新方式**：必须修改 HTML 中的硬编码数组。

```
每日 MD 文件 (9 BU)
    │

    │
reports/YYYY-MM-DD.json  (每日早报)
```

### 8.2 数据来源

| 优先级 | 数据来源 | 说明 |
|--------|---------|------|
| **第一优先** | `var EMBEDDED_POLICIES = [...]` | 内嵌在 HTML 第 388 行，混合了每日政策 + 静态 12 条核心政策 |
| **回退** | `policies_source/policies_clean.json` | 124KB，当 EMBEDDED_POLICIES 不存在时 fetch |
| **嵌入版** | `embedded/policy_center_embedded_data.js` | `window.__EMBEDDED__` 对象，含 report + policies |

### 8.3 政策数据来源分类

1. **每日政策** (`_isDaily: true`): 从 9 BU 早报的 `dim === "政策"` 板块提取，`md_to_json.py` 的 `build_policies_from_reports()` 自动累积
2. **静态核心政策** (`_isDaily: false`, id: "1"-"12"): 由 `reports/gen_policies.py` 硬编码生成，涵盖中央/地方/海外重大政策

### 8.4 更新方式

```
1. 每日早报生成 (D:\buddy\md_to_json.py)
   └── 自动调用 build_policies_from_reports()
   └── 新政策追加到 reports/policies.json（按 date+title[:50] 去重）

2. 生成嵌入版
   cd D:\trae\AI Daily report
   python embedded\gen_embedded_all.py
   └── 将 policies.json 注入 policy_center_v4.html 的 EMBEDDED_POLICIES
   └── 同时更新 embedded/policy_center_embedded_data.js

3. 验证
   http://localhost:8888/policy_center_v4.html
```

### 8.5 页面加载逻辑

```javascript
// 优先使用内嵌数据
async function loadPoliciesJson() {
    const data = window.EMBEDDED_POLICIES;  // inline 7MB 数组
    return Array.isArray(data) ? data : [];
}

// 嵌入版有 fetch override (window.__EMBEDDED__)
// 拦截 fetch('reports/*.json') → 返回内嵌 report
// 拦截 fetch('policies.json') → 返回 window.__EMBEDDED__.policies
```

### 8.6 政策类型分类规则（中央/地方/海外）

分类逻辑位于 `D:\buddy\md_to_json.py` 的 `build_policies_from_reports()` 函数中，**纯关键词匹配，无人工审核环节**。

#### 分类流程

```
默认 → "central" (中央)
  ↓ 检查正文是否包含地方关键词 → 匹配 → "local" (地方)
  ↓ 不匹配 → 检查正文是否包含海外关键词 → 匹配 → "overseas" (海外)
  ↓ 不匹配 → 保持 "central"
```

优先级：中央 → 地方 → 海外。一旦匹配到"地方"，就不会再判为"海外"。

#### 关键词清单 (`TYPE_KEYWORDS`)

| 类型 | 关键词 |
|------|--------|
| **中央** (`central`) | 国务院、工信部、发改委、能源局、商务部、财政部、生态环境部、国家、四部门、六部门、八部门、锂电池、动力电池、储能电池 |
| **地方** (`local`) | 省、市、区、县、地方、江苏、浙江、广东、山东、四川、北京、上海、深圳、广州 |
| **海外** (`overseas`) | 欧盟、美国、欧洲、德国、日本、韩国、IRA、EU |

#### 关键实现细节

- 匹配对象是政策**正文内容**（`item.get('content', '')`），不是标题
- 判断方式：`any(kw in content for kw in keywords)`
- `_isDaily: true` 的政策来自 9 BU 早报 `dim === "政策"` 板块
- `_isDaily: false` 的 12 条静态核心政策由 `reports/gen_policies.py` 硬编码生成，`type` 字段手动指定
- **无独立的分类运维文档**，此分类规则此前仅存在于代码中

### 14.6 常见错误与修复

| 错误现象 | 原因 | 解决方案 |
|---------|------|---------|
| 锂辉石图表数据停留在旧日期 | 数据源格式变更，过滤条件未更新 | 检查 `grade`/`origin` 是否匹配最新数据格式 |
| 锂云母图表显示多个品种锯齿状 | 数据文件包含多个品种未过滤 | 清理历史数据，只保留2.5%品种 |
| 锂辉石/锂云母图表显示 `"元/吨"` 但价格数量级错误 | 单位标错（锂辉石应是美元/吨） | 修正数据文件 `unit` 字段，锂辉石为 `"美元/吨"` |
| 自动化任务执行失败 | 脚本文件丢失或未提交git | 从git恢复或重建脚本，确保提交到git |
