## 一、早报系统常见错误代码速查

> 来源：`运维框架.md` Section 7

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `Failed to load resource: 404 (File not found) reports/2026-05-24.json` | `getLatestReportForBU()` 走网络请求 | 修复函数，优先使用嵌入数据 |
| `sectionsToHTML returned, html.length=0` | `dim` 值不匹配 | 确保 `dim` 是英文 |
| `buildBriefSection: no topnews section found` | `buildBriefSection()` 未找到 `topnews` section | 修改函数，从 `items` 读取 |
| 今日关注显示"暂无内容" | `section.content` 为空 | 修改 `buildBriefSection()` 读取 `items` |
| 市场/价格显示为卡片 | 未识别价格类条目 | 在 `sectionsToHTML()` 中添加 `market` 特殊处理 |

## 二、数据库与看板系统已知问题

> 来源：`DATABASE_UPDATE_WORKFLOW.md` Section 11

| 问题 | 影响 | 优先级 | 解决方案 |
|------|------|--------|---------|
| `_update_charts_data.py` 仅覆盖 LFP + Carbonate | Automotive、Recycling 看板缺乏自动更新 | 高 | 扩展脚本支持 Automotive、Recycling |
| LIB、Automotive、Recycling 缺少 `all_data.json` | 如果 JS 加载失败，数据库页面无回退数据 | 中 | 生成对应的 `all_data.json` 文件 |
| LFP 使用无前缀文件名 `embedded_data.js` | 易与 `embedded/embedded_data.js` 混淆 | 中 | 重命名 LFP 的文件为 `lfp_embedded_data.js` |
| `embedded/embedded_data.js` 与根目录不同步 | 8089 可能展示旧数据 | 中 | 每次更新后同步到 `embedded/` 目录 |
| JS 变量名不统一 (EMBEDDED_DATA vs ELECTROLYTE_DATA vs TERNARY_DATA) | 维护时需要记忆不同变量名 | 低 | 统一为 `EMBEDDED_DATA` |
| 无 Excel → embedded JS 的自动化生成脚本 | 数据更新依赖外部工具/手动操作 | 低 | 开发自动转换脚本 |

### 3.1 早报ZIP自动下载与更新任务（automation-1779699711838）

**故障日期**：2026-05-29

**故障现象**：
- Token 过期：`feishu_user_token.json` 中 `refresh_token` 已失效（`invalid_grant: The refresh token has been revoked`）
- 可兰素数据异常：飞书上的 `可兰素事业部-2026-05-29.json` 文件JSON格式有误（pos 2954处有未转义引号），无法解析

**解决方案**：
- Token 问题：用户重新完成 OAuth 授权，提供授权码换取新的 `access_token`
- 可兰素数据：使用2026-05-27数据作为备用（复制为 `reports/2026-05-29-kls.json`）

**改进建议**：
1. 修改 `download_feishu_docx_as_json.py` 支持直接下载 JSON 文件（不仅是 docx）
2. 使用 `tenant_access_token`（应用级，不过期）代替 `user_access_token`（用户级，需刷新）
3. 增加可兰素数据校验，提醒 LONGxia 重新上传

### 3.6 热点新闻生成自动化任务（hot-news-generator）

**故障日期**：2026-05-29

**故障现象**：
- Git push失败：执行 `git push` 时无法连接到 `github.com:443`（Failed to connect to github.com port 443）

**原因分析**：
- 可能是网络限制（企业防火墙）或临时网络问题
- 非代码bug，是运行时环境问题

**解决方案**：
- 数据生成和本地提交已完成，仅需用户手动推送或检查网络设置
- 如果持续失败，可考虑使用镜像仓库或VPN

**改进建议**：
- 在自动化任务中增加网络检测，如果push失败则等待5分钟后重试
- 或者将push操作分离为独立任务，避免阻塞数据生成

### 3.8 锂辉石锂云母价格数据自动更新任务（automation-1779788037620）

**故障日期**：2026-05-27

**故障现象**：
- `update_lithium_lepidolite_history.py` 缩进bug导致数据未添加：运行成功但历史数据条数未增加

**改进建议**：
1. 增加数据验证步骤：运行后检查历史数据条数是否增加
2. 使用单元测试覆盖关键逻辑

## 四、排障流程图

### 4.1 早报生成失败排障流程

```
早报生成失败
  ↓
检查 error log（自动化任务输出）
  ↓
├─ Token过期？ → 重新OAuth授权，更新 feishu_user_token.json
├─ 数据格式错误？ → 检查 MD 文件格式，修复后重新运行
├─ 脚本丢失？ → 从备份恢复或重建脚本
└─ 其他错误？ → 查看 运维框架.md Section 7 错误代码表
```

### 4.2 数据库页面显示异常排障流程

```
数据库页面显示异常（data_v2.html）
  ↓
检查浏览器 F12 Console
  ↓
├─ JS加载失败？ → 检查网络连接，确认 embedded_data.js 存在
├─ 数据过时？ → 检查 embedded_data.js 的 update_time，确认是否已更新
├─ 回退数据也失败？ → 检查 all_data.json 是否存在（部分数据库缺少）
└─ 其他问题？ → 查看 DATABASE_UPDATE_WORKFLOW.md Section 11 已知问题
```

### 4.3 看板图表不更新排障流程

```
看板图表不更新（charts.html）
  ↓
判断数据模式（动态 vs 硬编码）
  ↓
├─ 动态模式（Electrolyte/LIB/Ternary）→ 检查 embedded_data.js 是否已更新
├─ 硬编码模式（LFP/Carbonate/Automotive/Recycling）
│    ├─ LFP/Carbonate → 运行 python _update_charts_data.py
│    └─ Automotive/Recycling → 手动编辑 HTML 中的 var 数组
└─ 其他问题？ → 查看 DATABASE_UPDATE_WORKFLOW.md Section 5 更新方式
```
