# 龙蟠科技早报生成器 · 使用说明

## 适用场景

每日生成 9 个事业部早报 JSON，配套 WebSearch 搜索 + HTML 可视化。

---

## 快速启动（Workbuddy / Claude Code）

### 第一步：生成今日搜索任务

在 Workbuddy 对话框输入：

```
python generate_html_report.py --gen-tasks
```

Workbuddy 会输出 5 个搜索任务清单，照着执行 WebSearch。

### 第二步：汇总搜索结果

把 5 个任务的全部结果汇总为 JSON 数组（每条包含 `title`、`content`（100字以上）、`url`、`date`（YYYY-MM-DD）、`source`），保存到：

```
D:/trae/AI Daily report/search_results.json
```

### 第三步：生成早报

```
python generate_html_report.py
```

报告输出到：`D:/trae/AI Daily report/reports/YYYY-MM-DD.json`

---

## 快速启动（Trae）

完全相同的三步，只是命令末尾加 `--terminal trae`：

```
# 第一步
python generate_html_report.py --gen-tasks --terminal trae

# 第二步：结果保存到 ./search_results.json

# 第三步
python generate_html_report.py --terminal trae
```

---

## 可选参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--terminal` | 切换终端预设：`workbuddy`（默认）/ `trae` / `custom` | `--terminal trae` |
| `--input 路径` | 指定搜索结果 JSON 路径（覆盖默认） | `--input ./my_results.json` |
| `--input-dir` | 指定结果文件所在目录（配合 `custom` 使用） | `--input-dir ./data` |
| `--report-dir` | 指定报告输出目录 | `--report-dir ./output` |
| `--configs-dir` | 指定 BU 配置文件目录 | `--configs-dir ./configs` |
| `--window 7` | 扩大时间窗口到 T-7（用于搜索稀少时） | `--window 7` |
| `--inspect` | 仅预览输入文件摘要，不生成报告 | `--gen-tasks --inspect` |
| `--open` | 生成后自动打开浏览器预览 | `--open` |

## 常见问题

**Q: 搜索结果太少，早报内容空怎么办？**
用 `--window 7` 扩大时间窗口到 T-7：
```
python generate_html_report.py --gen-tasks --window 7
```

**Q: 想先看看输入数据对不对？**
加 `--inspect` 只预览，不生成报告：
```
python generate_html_report.py --inspect
```

**Q: 脚本修改后生效了没？**
每次执行生成任务时，Workbuddy 读取的是本地的 `generate_html_report.py`，确认使用的是最新版本。
