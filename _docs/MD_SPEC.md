# MD 文件规范与 JSON 输出格式

> 适用对象：生成早报数据的 AI 模型 / 手动编辑 MD 的用户
> 重点：确保生成的 MD 文件能被 md_to_json.py 正确解析

---

## 一、MD 文件格式规范

### 标准格式（推荐）

每个 MD 文件对应一个 BU，包含以下章节：

```markdown
【润滑油事业部】

## 🔥 企业动态          ← 或 ## ⚔️ 企业动态（兼容）
1. **宁德时代推出新一代神行超充电池**
   宁德时代发布全球首款采用磷酸铁锂材料并可实现大规模量产的4C超充电池，
   充电10分钟可续航400公里。（来源：宁德时代官微，2026-04-28）

## 💻 技术/产品
1. **比亚迪第二代刀片电池能量密度提升**
   第二代刀片电池单体能量密度达到190Wh/kg，系统能量密度超过160Wh/kg，
   续航里程提升至700km以上。（来源：比亚迪官网，2026-04-27）

## 📊 市场/价格
1. **磷酸铁锂价格本周上涨**
   动力型磷酸铁锂现货价格报42000元/吨，较上周上涨3.2%。（来源：鑫椤数据，2026-04-28）

## 📜 政策/行业
1. **工信部发布锂离子电池行业规范条件**
   工信部对锂电池企业的产能利用率、研发投入等提出新要求，推动行业高质量发展。
   （来源：工信部官网，2026-04-26）

## 🏗 项目/招标
1. **四川时代宜宾基地三期项目投产**
   宁德时代四川时代宜宾基地三期项目正式投产，总产能达100GWh。
   （来源：四川日报，2026-04-28）

## 💡 小结
4月XX日XX行业核心看点：
- 要点1
- 要点2
- 要点3
- 要点4
```

### 兼容格式 A（两种条目格式均可）

```markdown
## 政策/行业
1. **标题1**
正文内容。

2. **标题2**
正文内容。
```

### 兼容格式 B（数字在标题开头）

```markdown
## 政策/行业
**1. 标题1**
正文内容。
```

### Emoji 彩色字符注意

彩色 emoji（如 `🏗️`）是**两个 Unicode 码点**：
- U+1F3D7（基础 emoji）
- U+FE0F（变体选择符 VS16）

md_to_json.py 的正则必须包含两者：
```python
re.sub(r'^[\U0001F000-\U0010FFFF\U0000FE0F ]+', '', line)
```
- `\U0001F000-\U0010FFFF`：补充平面 emoji
- `\U0000FE0F`：VS16（**必须8位，写 `\U000FE0F` 会报错**）

## 二、章节标题映射

md_to_json.py 将 MD 章节映射到 JSON 维度：

| MD 章节关键词 | JSON 字段 |
|-------------|----------|
| `企业动态` / `⚔️` | 竞品 |
| `技术` / `产品` | 前沿 |
| `市场` / `价格` | 市场 |
| `政策` / `行业` | 政策 |
| `项目` / `招标` | 客户 |
| `💡 小结` | 小结 |

**注意**：只有常州锂源/龙蟠时代/山东美多/三金锂电/法恩莱特（锂电5BU）强制5维；
润滑油/可兰素/迪克化学/铂源催化（非锂电BU）维度可自定义。

## 三、条目格式规范

每条新闻条目必须包含以下信息：

```
标题（含关键词/关键数据）
正文内容
（来源：媒体名称，YYYY-MM-DD）
```

### 优先级标注（可选）

在标题行添加 `【P0】` / `【P1】` / `【P2】`：

```markdown
【P0】**宁德时代推出4C超充电池**
正文...
```

md_to_json.py 将 `【P0】` 提取为 `priority: "P0"`。

## 五、BU_MAP 配置（md_to_json.py）

```python
BU_MAP = {
    "lhy":  "润滑油事业部",
    "kls":  "可兰素事业部",
    "czly": "常州锂源事业部",
    "lpsd": "龙蟠时代事业部",
    "sdmd": "山东美多事业部",
    "sjld": "三金锂电事业部",
    "bych": "铂源催化事业部",
    "felt": "法恩莱特事业部",
    "dhx":  "迪克化学事业部",
}
```

MD 文件名**必须包含**上述中文事业部名（关键字匹配，不需完整）。

## 六、md_to_json.py 解析关键逻辑

### 双重条目格式兼容

```python
# 双遍 re.split，取并集去重
for pat in (r'\n(?=\*\*)', r'\n(?=\d+\. )'):
    parts = re.split(pat, raw)
```

### 条目优先级提取

```python
priority_match = re.search(r'【([PABC]\d)】', title)
# 或从标题关键词推断：P0（政策重大/竞品新品）→ P0
```

### 时间窗口过滤

`WINDOW_START` 和 `WINDOW_END` 控制内容回溯范围：
- 默认：T-2 到 T-1（或 T-2 到 T）
- 例：`WINDOW_START = "2026-04-26"` `WINDOW_END = "2026-04-27"` → 内容取4月26-27日的新闻

## 七、常见解析错误

| 现象 | 原因 | 解决 |
|------|------|------|
| 某BU只有4条（应为9条） | 条目格式不兼容（`N. **标题`** vs `**N. 标题`**） | 改用统一格式 |
| 章节标题解析为空 | emoji VS16 码点未处理 | 检查正则 `\U0000FE0F` |
| 内容被截断 | 条目之间没有空行 | 每个条目之间空一行 |
| 事业部名匹配失败 | MD 文件名不含 BU_MAP 中的中文 | 确认文件名包含事业部中文名 |
