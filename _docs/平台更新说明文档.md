## 2. 早报系统更新逻辑

### 2.1 数据源

- **输入**：9个BU的 Markdown 格式早报文件
- **存放路径**：`C:\Users\1\Downloads\YYYY-MM-DD-9bu-reports\YYYY-MM-DD\`
- **命名格式**：`{BU中文名}_早报_YYYY-MM-DD.md`

### 2.2 生成脚本

**文件位置**：`D:\buddy\md_to_json.py`

**配置项**（脚本顶部修改）：
```python
SRC_DIR  = Path(r"C:\Users\1\Downloads\2026-05-08-9bu-reports\2026-05-08")
OUT_DIR  = Path(r"D:\trae\AI Daily report\reports")
DATE_STR = "2026-05-08"          # 早报日期
WINDOW_START = "2026-05-06"      # 报告窗口开始
WINDOW_END   = "2026-05-07"      # 报告窗口结束
```

### 2.3 数据转换流程

```
MD 文件 → 解析 6 维度 → JSON 结构化 → 输出到 reports/
```

**6 维度映射**（MD → JSON）：

| MD 章节 | JSON 维度 key | 说明 |
|---------|--------------|------|
| 今日要报 | topnews | 集团汇总弹窗展示 |
| 市场/价格 | 市场 | 价格行情、供需数据 |
| 政策/行业 | 政策 | 政策法规、行业动态 |
| 企业动态 | 竞品 | 竞争对手动态 |
| 技术/产品 | 前沿 | 技术突破、新产品 |
| 项目/招标 | 客户 | 客户项目、招标信息 |

### 2.4 每日更新命令

```bash
# 1. 放置 MD 文件到下载目录
# 2. 修改 md_to_json.py 顶部的日期配置
# 3. 执行生成
cd D:\buddy && py -3.12 md_to_json.py

# 4. 生成 8089 嵌入版
cd D:\trae\AI Daily report && py -3.12 embedded\gen_embedded_all.py
```

### 2.5 数据校验

生成后会自动校验：
- 每个 BU 总条目数 ≥ 5
- 每个维度至少有数据
- 每条内容 ≥ 30 字
- 每条必须有 source 和 date

### 2.6 输出文件

- `reports/YYYY-MM-DD.json` — 主数据文件
- `reports/index.json` — 索引文件（可用日期列表）
- `reports/market_lc.json` — 碳酸锂市场数据
- `reports/market_lfp.json` — 磷酸铁锂市场数据

### 6.2 数据格式（EMBEDDED_DATA 模式）

以磷酸铁锂为例：

```javascript
// embedded_data.js
window.EMBEDDED_DATA = {
  "supply_demand": [...],      // 供需数据
  "price_history": [...],      // 价格历史
  "capacity": [...],           // 产能数据
  "output": [...],             // 产量数据
  "import_export": [...],      // 进出口数据
  "inventory": [...],          // 库存数据
  "cost_profit": [...],        // 成本利润
  "upstream": [...],           // 上游原材料
  "downstream": [...],         // 下游需求
  "policy": [...],             // 政策数据
  "company": [...],            // 企业动态
  "technology": [...]          // 技术进展
```

### 6.3 数据录入流程

1. **收集数据**：从行业网站、报告、数据库获取原始数据
2. **整理为 CSV/Excel**：按数据库模板整理
3. **转换为 JSON**：使用脚本或在线工具转为 JSON 数组
4. **更新 embedded_data.js**：将 JSON 数据赋值给 `window.EMBEDDED_DATA`
5. **验证展示**：打开对应 HTML 页面验证表格/图表渲染
6. **同步到嵌入版**：运行 `gen_embedded_all.py`

### 8.2 生成流程

```
加载数据文件
    ↓

    ↓
处理各类页面（A/B/C/D 类）
    ↓
```

### 8.3 页面分类处理

| 类别 | 页面 | 处理策略 |
|------|------|----------|
| **A** | index_v3.html, dept-archive.html, industry_news.html | 注入 fetch override + window.__EMBEDDED__ |

| **C** | policy_center_v4.html | 替换 fetch 调用为内联数据 |
| **D** | 其他静态页面 | 直接复制 |

### 8.4 关键机制：fetch Override

```javascript
// 拦截所有 reports/ 请求，返回内嵌数据
window.fetch = function(url) {
  if (url.includes('market_lc.json')) 
    return Promise.resolve({ok:true, json:()=>window.__EMBEDDED__.marketLc});
  if (url.includes('reports/YYYY-MM-DD.json'))
    return Promise.resolve({ok:true, json:()=>window.__EMBEDDED__.report});
  // ...
};
```

```html
<meta name="html-version" content="20260508f">
```

**版本号规则**：
- 日期部分：`YYYYMMDD`
- 当日迭代：`a, b, c, d, e, f...`
- 每次修改后必须递增（强制刷新用户缓存）

### 11.3 回滚流程


- **只维护 8888 + 8089 两个端口**，不在其他地址更新
