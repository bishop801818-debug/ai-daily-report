## 附录：关键设计决策

### A. 为什么使用原生 JavaScript?

- **零依赖**: 无需 npm、webpack 等构建工具
- **即开即用**: 双击 HTML 文件即可运行
- **易于维护**: 所有代码在一个文件中
- **学习成本低**: 开发者容易理解和修改

### B. 为什么使用 JSON 动态加载？

- **数据分离**: 内容与结构分离
- **易于更新**: 只需更新 JSON 文件
- **支持多日期**: 自动读取历史数据
- **降级方案**: JSON 加载失败时使用硬编码数据

### C. 动画设计理念

- **微交互**: 所有悬停都有反馈
- **流畅过渡**: 使用 cubic-bezier 缓动
- **性能优先**: 使用 transform 和 opacity
- **视觉层次**: 阴影和位移创造深度

### D. 响应式设计

- **Grid 布局**: 自动适应屏幕宽度
- **Flexbox**: 灵活的弹性布局
- **媒体查询**: 768px 断点
- **移动端优化**: 卡片单列显示

### 给其他模型的指令

1. **完全复制此页面**需要:

   - 在 `deptList` 数组添加新 ID
   - 在 HTML 添加新卡片
   - 在 `reportData` 添加对应数据
   - 或在 JSON 中添加新 department

4. **调整动画速度**:
   - 搜索 `transition: all 0.3s`
   - 修改时间值 (如 0.3s → 0.5s)

5. **修改布局**:
   - 调整 `.business-grid` 的 `minmax(400px, 1fr)`
   - 修改卡片最小宽度
