## 数据处理流程

### 1. 合并脚本
使用 `merge_search_results.py` 脚本进行数据合并：
```python
python merge_search_results.py
```

### 2. 格式转换
将两种不同格式的数据转换为统一的扁平列表格式：
```json
{
  "title": "新闻标题",
  "content": "正文摘要（100 字以上）",
  "url": "原文链接",
  "date": "YYYY-MM-DD",
  "source": "来源名称"
}
```

### 3. 去重处理
基于 title+url 组合进行去重，确保每条新闻唯一性。

### 4. 报告生成
使用 `generate_html_report.py` 脚本生成早报：
```powershell
python generate_html_report.py --terminal trae
```

## 关键技术特性

### 1. 旧闻复验三步法
- **标题层**：检测旧闻特征词（回顾、总结、此前等）
- **正文层**：抽取事件实际发生时间
- **来源层**：评估来源可信度（官方>媒体>转载）

### 2. 多级回落机制
- **L1**：T 至 T-2，高置信度，直接相关
- **L2**：T-3 至 T-5/T-7，中置信度，背景参考
- **L3**：AI 生成 1 句判断文案
- **L4**：结构性判断文案模板

## 下一步优化建议

### 1. 关键词匹配优化
- 当前部分事业部匹配率较低（如 dhx 为 0）
- 建议扩展关键词库，增加同义词和别名

### 2. 内容质量提升
- 部分条目 content 字段长度不足 100 字
- 建议优化搜索提示词，要求返回更详细的摘要

### 3. 时间窗口调整
- 当前使用 T 至 T-2 标准窗口
- 可根据业务需求调整为 T 至 T-5 或 T 至 T-7

### 4. 来源优先级

- 对高优先级来源给予更高权重
